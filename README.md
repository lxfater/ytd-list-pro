## 功能特性

- **订阅同步**：一键读取当前登录 YouTube 账号的完整订阅列表，保存在本地。
- **无限分类**：自由创建分类，自定义名称、颜色和图标。
- **原生侧边栏**：分类以原生观感嵌入 YouTube 侧边栏，可替换冗长的原订阅列表，随时切换回去。
- **一键归类**：在视频页和频道主页的订阅按钮旁有「加入分类」按钮，看到哪个博主顺手就能归类；按钮会直接显示频道当前所属的分类。
- **全局搜索**：输入关键字即可跨全部分类搜索频道，支持多关键字（空格分隔、不限顺序），结果标注所属分类。
- **批量管理**：管理面板支持多选、批量移动、拖拽整理和多种排序（最近关注、名称等）。
- **多账号隔离**：每个 YouTube 账号的分类数据独立存储，切换账号互不干扰；旧版本的数据会在升级后自动迁移给第一个使用的账号。
- **新视频提醒**：后台定时轮询已订阅频道的公开 RSS，有新视频发布时会在侧边栏头像右上角显示一个小红点，点开该频道后自动清除。只追踪功能启用之后发布的新视频，不代表 YouTube 官方的"未观看"状态。
- **导入 / 导出**：分类关系可一键导出为 CSV（Excel 可直接打开），也可以从 CSV 批量导入。
- **数据本地化**：全部数据保存在浏览器 `chrome.storage.local`，没有后端服务器。

## 使用教程

![使用教程](assets/help.png)

跟着图中的 ①②③ 三步上手：

1. **打开管理面板**（图中 ①）：安装后，YouTube 左侧边栏顶部会出现「原始 / 分类 / 管理」切换按钮，点击「管理」弹出分类管理面板。
2. **新建分类并整理频道**（图中 ②）：点击「新建」创建自己的分类，勾选频道后点「移动」（或直接拖拽）把它们归入对应分类；顶部搜索框可以跨全部分类查找频道。
3. **刷新频道**（图中 ③）：首次使用先点「刷新频道」，把当前账号的订阅同步进来。**之后每次关注了新博主，列表里看不到时，点一下「刷新频道」就会出现。**

整理好之后，左侧边栏切换到「分类」模式，点击任意分类即可展开浏览其中的频道——从此告别一长串杂乱的订阅列表。日常使用还有更快的方式：在任意视频页或频道主页，订阅按钮旁会出现「加入分类」按钮，点开选一个分类即可完成归类，不用打开管理面板。

## 安装

### 方式一：下载 Release 安装包（推荐，无需构建）

不用装 Node、不用跑命令，直接用打包好的版本：

1. 前往 [Releases 页面](https://github.com/lxfater/ytd-list-pro/releases/latest)，下载最新的 `ytd-list-pro-vX.Y.Z.zip` 并解压。
2. 打开 `chrome://extensions/`（Edge 为 `edge://extensions/`），开启右上角「开发者模式」。
3. 点击「加载已解压的扩展程序」，选择解压后的文件夹。
4. 打开 [youtube.com](https://www.youtube.com)，按上面的使用教程同步订阅。

### 方式二：从源码构建

```bash
npm install
npm run build
```

构建完成后，同样按上面第 2-4 步操作，「加载已解压的扩展程序」时选择本项目的 `dist` 目录。

## 数据导出 / 导入

管理面板工具栏提供「导出」和「导入」两个按钮。

**导出**：把全部分类和频道下载为 CSV 文件（UTF-8 带 BOM 编码，用 Excel 双击打开中文不乱码）。

**导入**：选择一个相同结构的 CSV 文件即可批量建立分类关系。文件第一行是表头（可省略），之后每行一个频道，共三列：

| 分类 | 频道名称 | 频道链接 |
| --- | --- | --- |
| AI大师 | Matt Wolfe | https://www.youtube.com/@mreflow |
| 编程 | Tech With Tim | https://www.youtube.com/@TechWithTim |
| 未分类 | | https://www.youtube.com/channel/UCxxxxxxxx |

各列规则：

- **分类**（必填）：分类名称。不存在的分类会自动创建；写「未分类」则频道进入未分类列表。
- **频道名称**（可留空）：留空时会从链接自动推断，之后刷新订阅时会补全为真实名称。
- **频道链接**（必填）：支持 `https://www.youtube.com/@handle` 和 `https://www.youtube.com/channel/UC...` 两种形式。
- 已存在的频道（按频道 ID 或 handle 匹配）会**移动**到目标分类，不会重复创建；无法识别的行会被跳过并在状态栏提示行号。

提示：先用「导出」生成一份 CSV 当模板改，是最不容易出错的导入方式。

## 隐私说明

- 本扩展**不包含**任何真实用户的订阅数据。
- 订阅数据通过你浏览器中已登录的 YouTube 会话在 YouTube 页面内读取，仅保存在本地浏览器存储中。
- 项目没有后端服务，不会把订阅列表上传到任何第三方服务器。
- 内容脚本读取 YouTube 页面配置，仅用于在 YouTube 自己的域内调用其内部订阅接口；任何 Cookie 值都不会被记录、提交或发送到 YouTube 之外。
- 新视频提醒功能会向 YouTube 官方公开的 RSS 接口（`youtube.com/feeds/videos.xml`）发起请求，不需要登录、不携带 Cookie，仅用于比对频道是否有新视频发布，结果只保存在本地。
